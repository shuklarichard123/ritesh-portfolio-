Slide 02 · Background
Problem & Solution

⚠️ The Problem — Current State

  • IP whitelisting requests raised via email — no standardisation or audit trail
  • No unique tracking ID — impossible to follow up on a specific request
  • Missing critical details (IP, port, protocol, environment) cause delays
  • No record of business justification or named approver
  • Network team manually chase requesters for missing information
  • No visibility into pending, approved or rejected requests

✅ The Solution — New System

  • Structured web form enforces all required fields before submission
  • Every request gets a unique UUID — shareable for tracking and follow-up
  • Captures: IP/CIDR, port, protocol, environment, duration, application name
  • Mandatory business justification and named approver on every request
  • DynamoDB provides permanent, queryable audit trail of all requests
  • Admin view lets network team list and filter requests instantly

Paste any Request ID to retrieve full details including justification, approver and all IP parameters.

Slide 06 · Data Storage
DynamoDB Data Model
AttributeTypeDescription
request_idPK · UUIDAuto-generated unique identifier for every request
usernameStringFull name of the requester (GSI partition key)
requested_dataStringOrganisation or team name of the requester
input_parametersMap (JSON)Nested object containing all IP/access fields below
↳ ip_addressStringIPv4 address or CIDR block (e.g. 10.0.0.0/16)
↳ portStringPort number or range (e.g. 443 or 8080-8090)
↳ environmentStringTarget environment: Production / Staging / Dev / UAT
↳ access_durationString7 Days / 30 Days / 90 Days / Permanent
↳ protocolsListArray of protocols: HTTPS, HTTP, SSH, TCP, UDP
↳ business_justificationStringFree-text business reason for the access request
↳ approver_nameStringName of the approving manager or authority
↳ requester_emailStringEmail address of the requester for follow-up
created_atTimestampISO 8601 UTC timestamp of submission

🔑 Primary Key

request_id (UUID v4) is the partition key. Globally unique — no hot partitions. Enables O(1) lookup by ID.

📇 Global Secondary Index

username-index on the username attribute. Allows admin to query all requests by a specific requester instantly.

💰 Billing Mode

PAY_PER_REQUEST — no provisioned capacity. Zero cost when idle. Scales automatically with traffic.

📦 Storage

Each record is ~1–2 KB. 25 GB free tier storage — can store millions of requests at zero cost.

Slide 07 · API
REST API Endpoints
POST /prod/records Submit a new IP whitelisting request. Validates all required fields, generates a UUID request_id and writes to DynamoDB. Body: JSON with username, requested_data, input_parameters
GET /prod/records Retrieve all whitelisting requests. Returns list sorted newest first. Supports optional username filter via query string. ?username=ritesh.shukla (optional)
GET /prod/records/{request_id} Fetch a single request by its unique ID. Returns full record including all IP parameters, justification and approver details. Path: UUID request_id
